    def place_subtree_horizontal(root, spine_x, spine_y, snap_threshold, node_count,

    Algorithm (spine walk from root leftward along input[0] chain):

    1. `current_prefs = current_prefs or node_layout_prefs.prefs_singleton`
    2. Resolve scheme_multiplier: `if scheme_multiplier is None: scheme_multiplier = current_prefs.get("normal_multiplier")`
    3. `step_x = int(current_prefs.get("horizontal_subtree_gap") * scheme_multiplier)` — spine step is gap + node width (see formula in context)
    4. Build the spine list: walk input[0] chain from root, collect nodes until no input[0] exists. List is [root, input[0] of root, input[0] of that, ...] — root is rightmost (index 0), each successive element is one step left.
    5. Place each spine node:
       - `cur_x = spine_x - i * step_x - sum(n.screenWidth() for n in spine_nodes[:i])` (or track cur_x incrementally)
       - `cur_y = spine_y` (adjusted by cumulative_kink for each step — see mask kink below)
       - Call `spine_node.setXpos(cur_x)`, `spine_node.setYpos(cur_y)`
    6. For each spine node, place side inputs (input[1], input[2], etc. — non-input[0] slots) ABOVE the spine node using existing vertical formulas:
       - For each side input at slot s: `side_node = spine_node.input(s)` (skip None and nodes not visible / out of filter)
       - Mask side-swap suppression: do NOT reorder mask to the left — all side inputs go above the spine node (negative Y direction)
       - Place side input: `side_y = cur_y - vertical_gap_between(side_node, spine_node, snap_threshold, scheme_multiplier) - side_node.screenHeight()`; `side_x = _center_x(side_node.screenWidth(), cur_x, spine_node.screenWidth())`
       - For subtrees rooted at side inputs, call place_subtree() (the existing VERTICAL function) recursively to lay out the side input's upstream tree above it
    7. Mask kink: if a spine node has a mask input (detect via _is_mask_input()), compute the mask subtree height from compute_dims(mask_node, memo, snap_threshold, node_count, scheme_multiplier=scheme_multiplier). Add `mask_subtree_height + _subtree_margin(spine_node, mask_slot, node_count, mode_multiplier=scheme_multiplier)` to a running `cumulative_kink_y`. All downstream spine nodes (closer to root) are placed at `spine_y + cumulative_kink_y`. Walk the spine UPSTREAM-FIRST (from root's deepest ancestor down to root) so kinks accumulate in the right direction.

    Implementation note on walk direction: It is cleanest to first collect the full spine list [root, anc1, anc2, ...] (root at index 0, farthest ancestor at last index), then iterate in REVERSE order (farthest ancestor first) to accumulate kinks. Place each node as the walk returns toward root, so cumulative_kink_y grows from left (upstream) to right (downstream).

    CRITICAL: In horizontal mode, mask inputs go ABOVE the spine node, NOT to the left. Do NOT call _reorder_inputs_mask_last() with fan_active=True for the spine's side inputs — use plain input order or suppress fan reordering. Pass fan_active=False when determining side input order.

    CRITICAL (Nuke Y-axis): "above" = LOWER Y value. side_y is computed by subtracting from cur_y, not adding.
