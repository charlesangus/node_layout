<decisions_made>

- `_find_or_create_output_dot(root, consumer_node, consumer_slot, ...)` is the correct low-level function; it must NEVER be called with consumer_node == root (circular topology)
- New `_place_output_dot_for_horizontal_root` is the correct call site for all three layout functions (layout_upstream, layout_selected, _layout_selected_horizontal_impl)
- The dot should only be created when there is a real downstream consumer (non-output-dot node with root as direct input); if root is the final output, no dot is created
- Dot X position: centred on root (directly below root), NOT on consumer
- Dot Y gap: always loose_gap (loose_gap_multiplier * scheme * snap), NOT the compact same-tile-colour gap
- When layout_upstream finds a horizontal root via ancestor walk, the chain is anchored above the originally-selected downstream node: spine_y = E.ypos() - loose_gap - 12 - loose_gap - root.screenHeight()
