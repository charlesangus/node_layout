<context>
The original phase 11 implementation (two separate Upstream/Selected Horizontal
commands, side inputs placed above spine with simple setXpos/setYpos) did not match
what the user had discussed in a prior session. We completely redesigned it.

The new design is semantically different: "Layout Selected Horizontal" works by
taking whatever nodes the user has selected and treating them as the horizontal
spine. Non-selected inputs of spine nodes get full vertical layout. This is more
intuitive and flexible than the original "follow all of input[0] from a single root"
approach.

Two remaining issues were fixed at the end of the session:
1. BBox overlap: wide side subtrees were overlapping adjacent spine nodes. Fixed
   via pre-pass computing effective_widths[] and using them in cur_x advance.
2. Leftmost routing: the upstream input[0] of the farthest spine node was being
   placed above the spine, causing potential overlap. Now placed to the LEFT via
   a Dot node (mirrors the output Dot pattern at the root end).

Everything is committed. The phase needs live Nuke UAT before it can be signed off.
</context>
