    Four changes to node_layout.py:

    **Change 1 — compute_dims memo key (2 locations in compute_dims):**
    Add `layout_mode="vertical"` parameter to compute_dims signature. Extend memo key at both occurrences (the lookup and the store) to include layout_mode:
    `(id(node), scheme_multiplier, node_h_scale, node_v_scale, layout_mode)`
    All existing callers of compute_dims() omit layout_mode and receive the default "vertical" — no callers need updating for backward compatibility. The place_subtree_horizontal() caller (Plan 02) should pass layout_mode="horizontal".

    **Change 2 — layout_upstream() mode dispatch:**
    After `nuke.Undo.begin()` and after per-node scheme dict is built, read the root's stored mode:
    ```python
    root_stored_mode = node_layout_state.read_node_state(root).get("mode", "vertical")
    ```
    Add a branch: if root_stored_mode == "horizontal", call the horizontal implementation instead of the vertical compute_dims/place_subtree path. In the state write-back loop, write `stored_state["mode"] = "horizontal"` instead of `"vertical"` on the horizontal path. Write `"vertical"` on the vertical path as before.

    The horizontal dispatch in layout_upstream() should:
    - Call place_subtree_horizontal(root, root.xpos(), root.ypos(), snap_threshold, node_count, scheme_multiplier=root_scheme_multiplier, per_node_h_scale=per_node_h_scale, per_node_v_scale=per_node_v_scale, current_prefs=current_prefs, current_group=current_group)
    - Call _find_or_create_output_dot(root, snap_threshold, root_scheme_multiplier, current_group)
    - State write-back: stored_state["mode"] = "horizontal"
    - push_nodes_to_make_room (same as vertical path)

    **Change 3 — layout_selected() mode dispatch:**
    Same pattern as layout_upstream() — read mode from each root's stored state; dispatch to horizontal for roots with mode == "horizontal". State write-back writes "horizontal" on horizontal path.

    **Change 4 — Add layout_upstream_horizontal() and layout_selected_horizontal() entry points:**
    Add after layout_selected_loose() (around line 930). Both follow the established undo pattern exactly (global _TOOLBAR_FOLDER_MAP = None, _clear_color_cache(), current_group = nuke.lastHitGroup() as FIRST Nuke call, nuke.Undo.name(...), try/except/else).
