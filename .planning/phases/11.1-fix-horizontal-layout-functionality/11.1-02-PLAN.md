<objective>
Fix the two geometry bugs in node_layout.py that cause incorrect placement when a horizontal B-spine chain is embedded in a larger vertical tree.

Bug 1 fix: Pre-compute the full leftward extent of the spine before committing to spine_x, so the leftmost spine node lands right of the consumer with a clean gap — not overlapping it.

Bug 2 fix: Thread consumer_node Y into all three dot-position code paths so the output dot sits at the consumer's Y level (horizontal wire), not below the root (diagonal wire).

Purpose: Both bugs cause visually wrong layouts when the user re-lays out a vertical tree that contains a horizontal chain. The target geometry has the output dot centred vertically on the consumer node so the connecting wire is strictly horizontal, and the entire chain to the right of the consumer with no overlap.
Output: node_layout.py with 5 targeted line-level changes; TestLeftExtentOverlap and TestDotYAlignment turn GREEN.
</objective>
