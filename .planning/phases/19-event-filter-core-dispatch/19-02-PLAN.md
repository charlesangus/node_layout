- Test pattern reference: `tests/test_node_layout_overlay.py` (canonical), `tests/test_node_layout_prefs_dialog.py` (secondary)

## Tasks

### Task 1: Create test file with helpers and class/method existence tests

**Files:** `/workspace/tests/test_node_layout_leader.py`

**Action:**

Create the test file following the exact pattern from `test_node_layout_overlay.py`:

1. Module docstring listing what is verified:
   - LeaderKeyFilter class exists and subclasses QObject
   - eventFilter method is defined on LeaderKeyFilter
   - arm module-level function exists
   - _disarm module-level function exists
   - Dispatch table covers V, Z, F, C keys
   - Auto-repeat guard present (isAutoRepeat)
   - Mouse event cancellation present (MouseButtonPress)
   - Timer cancellation present (stop)
   - installEventFilter and removeEventFilter calls present

2. Imports: `ast`, `os`, `unittest` only (NO PySide6)

3. Helper constants and functions (matching overlay test pattern exactly):
   ```python
   LEADER_PATH = os.path.join(os.path.dirname(__file__), "..", "node_layout_leader.py")

   def _load_leader_source():
       """Return the source code of node_layout_leader.py as a string."""
       with open(LEADER_PATH, "r") as source_file:
           return source_file.read()

   def _parse_leader_ast():
       """Return the AST of node_layout_leader.py."""
       return ast.parse(_load_leader_source())
   ```

4. `TestLeaderFilterClassExists(unittest.TestCase)` -- verifies LEAD-02/LEAD-03 infrastructure:

   - `test_leader_key_filter_class_exists`: Parse AST, collect all `ClassDef` names, assert `"LeaderKeyFilter"` is present.

   - `test_leader_key_filter_inherits_qobject`: Find the `LeaderKeyFilter` ClassDef node, extract base class names (handle both `ast.Name` with `.id` and `ast.Attribute` with `.attr`), assert `"QObject"` is in the base names.

   - `test_event_filter_method_defined`: Find the `LeaderKeyFilter` ClassDef, walk its body for `FunctionDef` nodes, collect method names, assert `"eventFilter"` is present.

5. `TestModuleFunctionsExist(unittest.TestCase)` -- verifies arm/disarm exist:

   - `test_arm_function_exists`: Parse AST, collect top-level `FunctionDef` names (only direct children of the module, not nested), assert `"arm"` is present.

   - `test_disarm_function_exists`: Same approach, assert `"_disarm"` is present.
