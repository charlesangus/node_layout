    Structure:
    - Imports: PySide6.QtWidgets (QDialog, QDialogButtonBox, QFormLayout, QLineEdit, QVBoxLayout), node_layout_prefs
    - NodeLayoutPrefsDialog(QDialog):
      - __init__(self, parent=None): super().__init__(parent), setWindowTitle("Node Layout Preferences"), setMinimumWidth(400), _build_ui(), _populate_from_prefs()
      - _build_ui(): creates QVBoxLayout(self), QFormLayout, adds one QLineEdit per parameter (stored as self.{key}_edit), adds QDialogButtonBox(Ok | Cancel) with accepted -> _on_accept and rejected -> self.reject
      - QLineEdit fields and their form row labels (in this order):
          self.base_subtree_margin_edit      "Base Subtree Margin (px):"
          self.compact_multiplier_edit       "Compact Multiplier:"
          self.normal_multiplier_edit        "Normal Multiplier:"
          self.loose_multiplier_edit         "Loose Multiplier:"
          self.loose_gap_multiplier_edit     "Loose Gap Multiplier:"
          self.mask_input_ratio_edit         "Mask Input Ratio:"
          self.scaling_reference_count_edit  "Scaling Reference Count:"
        Do NOT add a field for tight gap (snap_threshold * 1) — it is not user-configurable per locked decision.
        Do NOT add a preset selector widget — presets are Phase 5 menu commands.
      - _populate_from_prefs(): reads from node_layout_prefs.prefs_singleton.get(key) and sets each QLineEdit text with str()
      - _on_accept(): wraps all int()/float() conversions in a single try/except ValueError block.
          base_subtree_margin and scaling_reference_count use int(); all others use float().
          Also validate base_subtree_margin > 0 and scaling_reference_count >= 1 (prevents ZeroDivisionError in sqrt formula). If any validation fails, return without calling self.accept() (dialog stays open).
          On success: call prefs.set() for each key, prefs.save(), self.accept().
    - Module-level show_prefs_dialog() function: instantiates NodeLayoutPrefsDialog() and calls dialog.exec()

    Use descriptive variable names throughout. No bare `prefs` shadowing the module name —
    use `prefs_instance = node_layout_prefs.prefs_singleton` inside methods.
