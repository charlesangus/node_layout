## Decisions Made

- **CONTEXT.md formula was wrong for Fix B:** CONTEXT.md specified rightmost = `m1.xpos() + (m1.screenWidth() + phase2_w) // 2` (centering formula for vertical subtrees). But place_subtree with all_side=True places root at anchor and steps side inputs rightward; actual rightmost = `anchor + phase2_w`. Using the centering formula underestimated the rightmost by `(phase2_w - m1.screenWidth()) / 2`, causing the clamp to be too lenient.
- **Strict-less-than boundary:** The BUG-B test uses `assertLess` (strict `<`), but the clamped anchor yields `b_max_x = chain_left - gap` exactly (boundary). Fixed by using `>=` condition and subtracting 1 pixel from the clamped anchor so `b_max_x = chain_left - gap - 1 < chain_left - gap`.
- **Fix A scoping:** `downstream_consumer` is only defined in the else-branch, but Fix A's `elif downstream_consumer is not None:` is only evaluated when `root is original_selected_root`, at which point the else-branch code has executed and `downstream_consumer` is in scope. No NameError.
