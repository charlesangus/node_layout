<context>
The core difficulty: `compute_dims` returns `(W, H)` where W is measured RIGHTWARD from the root node's x. There is no left-extension concept. A freeze block whose members extend left of the root has an asymmetric bbox that the layout engine doesn't understand natively.

The fix introduces `left_ovh` as the explicit left extension offset. The "alloc_left model" separates the band's left edge from the root node's x position: `root_x = alloc_left + left_ovh`. When `left_ovh = 0` (all non-freeze cases), the model is identical to the old code.

For centering (primary input above consumer): `_centered_alloc_left` centers `total_w` (the full block width) above the consumer, then root is placed `left_ovh` pixels into that band — so the whole block is visually centered.

For right-side inputs: alloc band starts at `current_alloc`, root placed at `current_alloc + left_ovh`, band advances by `child_dims[i][0] = total_w` to clear the block's full right edge.
</context>
