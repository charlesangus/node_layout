<context>
User requested a comprehensive review of how the layout algorithm works and how freeze
integrates with it. The algorithm is a clean two-pass system: compute_dims (bottom-up
bounding box accumulation) then place_subtree (top-down placement using accumulated sizes).

The root issue: compute_dims returned (W, H) where W was measured rightward from the root
node's x. Freeze blocks break this because the root can sit in the middle of the block,
not at the left edge. The prior fix used a side channel (freeze_block_sizes dict) passed
into place_subtree with _left_ovh()/_centered_alloc_left() closures adjusting all 5
X-placement branches.

The refactor embeds root_x_offset as the third element of the dims tuple so freeze blocks
participate in the same allocation model as everything else. For non-freeze nodes,
root_x_offset=0 and all behavior is identical to before the freeze feature existed.
</context>
