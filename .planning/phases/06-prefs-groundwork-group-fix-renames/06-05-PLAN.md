The fix from Plan 06-04 is already in place. nuke.thisGroup() is absent from
both layout_upstream() and layout_selected(). The `with current_group:` blocks
wrap all node creation in both functions. push_nodes_to_make_room() receives
current_group and scopes its node enumeration to current_group.nodes() when
inside a Group.

nuke.lastHitGroup() returns the Group node whose inline Group View panel was
most recently clicked into. At root (no Group View active), it returns nuke.root()
— the same safe fallback as the old thisGroup() call. Foundry documents this API
explicitly for menu commands that must be Group View aware.
