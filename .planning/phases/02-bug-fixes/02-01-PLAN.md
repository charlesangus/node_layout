From make_room.py (current buggy logic — x_amount/y_amount not always defined before use):
```python
def make_room(amount=1600, direction="up"):
    nodes_to_move = nuke.selectedNodes()

    if direction == "up":
        x_amount = 0
        y_amount = -amount
    elif direction == "down":
        y_amount = amount   # BUG: x_amount not set here
        x_amount = 0

    if direction == "left":   # NOTE: 'if', not 'elif' — evaluated independently
        y_amount = 0
        x_amount = -amount
    elif direction == "right":
        y_amount = 0
        x_amount = amount
    # After these conditionals, x_amount and y_amount are only guaranteed defined
    # when direction is "up". Other branches set them but if a new direction value
    # were passed, neither would be defined and the for loop below would crash.

    for node in nodes_to_move:
        node.setXpos(node.xpos() + x_amount)
        node.setYpos(node.ypos() + y_amount)
```
