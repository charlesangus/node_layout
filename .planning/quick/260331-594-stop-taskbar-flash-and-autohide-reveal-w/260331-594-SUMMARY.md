# Quick Task 260331-594: Stop Taskbar Flash and Autohide Reveal Summary

**One-liner:** Added `reparent()` to `LeaderKeyOverlay` that restores `Tool|FramelessWindowHint`, `WA_ShowWithoutActivating`, and `WA_TranslucentBackground` after `setParent()`, wired into `arm()` to fix Windows taskbar flash on subsequent leader mode activations.

## What Was Done

Qt's `setParent()` silently resets all window flags and widget attributes on the target widget. When `arm()` called `_overlay.setParent(dag_widget)` on re-invocation, `WA_ShowWithoutActivating` was lost — causing Windows to treat the next `show()` call as a foreground window request, triggering taskbar icon flash and autohide taskbar reveal.

### Changes

**node_layout_overlay.py** — New `reparent(new_parent)` method on `LeaderKeyOverlay`, placed between `__init__` and `_build_ui`:
- Calls `setParent(new_parent)` for the actual parent change
- Immediately re-applies `setWindowFlags(Qt.WindowType.Tool | Qt.WindowType.FramelessWindowHint)`
- Re-applies `setAttribute(Qt.WidgetAttribute.WA_ShowWithoutActivating)`
- Re-applies `setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground)`

**node_layout_leader.py** — In `arm()`, replaced:
```python
_overlay.setParent(dag_widget)
```
with:
```python
_overlay.reparent(dag_widget)
```
