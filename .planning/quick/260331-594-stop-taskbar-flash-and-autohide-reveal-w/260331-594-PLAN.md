<objective>
Fix Windows taskbar flash and autohide reveal when leader mode activates on subsequent invocations.

Purpose: Qt's setParent() resets all window flags and widget attributes. When arm() calls _overlay.setParent(dag_widget) on re-invocation, WA_ShowWithoutActivating is lost, causing Windows to treat the overlay's show() as a foreground window request — triggering taskbar icon flash and autohide reveal.

Output: A reparent() method on LeaderKeyOverlay that safely wraps setParent() with flag/attribute restoration, wired into arm().
</objective>

    ```python
    def reparent(self, new_parent):
        """Re-parent the overlay, restoring all window flags and attributes.

        Qt's setParent() resets ALL window flags and widget attributes on the
        widget.  This method wraps setParent() and immediately re-applies the
        flags that __init__ originally set, preventing Windows from treating a
        subsequent show() as a foreground window activation request (which
        triggers taskbar icon flash and autohide taskbar reveal).
        """
        self.setParent(new_parent)
        self.setWindowFlags(Qt.WindowType.Tool | Qt.WindowType.FramelessWindowHint)
        self.setAttribute(Qt.WidgetAttribute.WA_ShowWithoutActivating)
        self.setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground)
    ```
