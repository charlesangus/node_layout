<objective>
Fix 4 failing tests in test_scale_nodes_axis.py caused by setUp methods overwriting
_nl.nuke with the wrong nuke stub module.

Purpose: The setUp methods in TestRepeatLastScaleBehavior and TestExpandPushAway (and 2
other test classes) execute `_nl.nuke = sys.modules["nuke"]`. When another test file
(e.g., test_center_x.py) loads first and registers a nuke stub WITHOUT an Undo class,
sys.modules["nuke"] points to that incomplete stub. The setUp then overwrites _nl.nuke
(which was correctly set during module import with Undo) with the broken stub, causing
`nuke.Undo.name(...)` calls to fail with AttributeError.

Output: Patched test_scale_nodes_axis.py where setUp methods restore _nl.nuke from a
saved module-level reference rather than from sys.modules["nuke"].
</objective>
