<objective>
Fix leader key event filter to consume ShortcutOverride events during leader mode,
preventing Nuke's built-in shortcuts (e.g., C = ColorCorrect) from firing while the
leader key modal is active.

Purpose: Qt dispatches QEvent.ShortcutOverride BEFORE KeyPress. If the event filter
does not accept/consume ShortcutOverride, Qt matches the key against registered
shortcuts (Nuke's menu shortcuts). By the time the KeyPress event arrives and the
filter consumes it, Nuke has already dispatched the shortcut action. The fix is to
return True for ShortcutOverride events while _leader_active is True.

Output: Patched node_layout_leader.py, updated tests
</objective>
