# Quick Task 260331-537: Make Leader Popup Buttons Clickable — Summary

**One-liner:** Added `dispatch_key()` to leader module and `ClickableKeyCell(QWidget)` to overlay so every key cell triggers its leader action on mouse click, with hand cursor hover affordance.

## What Was Built

**node_layout_leader.py:**
- Added `_LETTER_TO_QT_KEY` module-level dict mapping all 10 letter strings to Qt key codes
- Added public `dispatch_key(key_letter)` function that mirrors eventFilter dispatch logic: single-shot keys call `_disarm()` then the handler; chaining keys hide the overlay and call the handler while keeping leader mode active

**node_layout_overlay.py:**
- Added `ClickableKeyCell(QWidget)` class before `LeaderKeyOverlay`
  - Constructor takes `key_letter` and `action_label`, builds badge + action label layout
  - Sets `Qt.CursorShape.PointingHandCursor` in `__init__` for hover affordance
  - Overrides `mousePressEvent` to call `node_layout_leader.dispatch_key(self._key_letter)` via inline import
- Refactored `_make_key_cell` from ~25 lines to a one-liner `return ClickableKeyCell(key_letter, action_label)`

**tests/test_node_layout_leader.py:**
- Added `TestDispatchKeyFunction` class with 3 tests: function exists, `_LETTER_TO_QT_KEY` exists, function body references `_disarm`

**tests/test_node_layout_overlay.py:**
- Added `TestClickableKeyCells` class with 5 tests: class exists, inherits QWidget, defines `mousePressEvent`, references `dispatch_key`, references `PointingHandCursor`
