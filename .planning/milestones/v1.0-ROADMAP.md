# Milestone v1.0: Quality & Preferences

**Status:** ✅ SHIPPED 2026-03-05
**Phases:** 1–5
**Total Plans:** 13

## Overview

Transformed node_layout from a working-but-fragile tool into a reliable, undoable, and configurable Nuke DAG layout plugin. This milestone addressed accumulated technical debt, corrected five user-visible layout bugs, added full undo support, built a persistent preferences system with PySide6 dialog, and delivered new compact/loose layout schemes plus shrink/expand scaling commands.

## Milestone Summary

**Key Decisions:**

- Follow Labelmaker prefs pattern for sibling plugin consistency
- Store prefs at ~/.nuke/node_layout_prefs.json to match Nuke convention
- Add custom knob to diamond-resolution Dots (safer than relying on hide_input)
- Cache preference lookups per layout operation only, not globally
- Use try/except/else (not finally) for undo group — nuke.Undo.end() in else, nuke.Undo.cancel() in except
- sqrt formula: int(base * multiplier * sqrt(node_count) / sqrt(reference_count)) — backward-compatible at reference_count=150
- scheme_multiplier resolves to normal_multiplier on first None check, not at every recursive level
- SHRINK_FACTOR=0.8, EXPAND_FACTOR=1.25 as module-level constants (not in prefs)
- Center-based offsets (screenWidth/2) in scale functions — Dot nodes move same fractional distance as regular nodes
- snap_min floor only in _scale_selected_nodes, not _scale_upstream_nodes

**Issues Resolved:**

- Input-0 centering was broken — nodes appeared offset to the side of their consumer
- Diamond Dot nodes were centered by side but not repositioned post-placement
- make_room() referenced undefined x_amount/y_amount in some branch combinations
- node_filter stored node IDs rather than node objects (stale after script modifications)
- Ctrl+Z after layout moved nodes one-by-one rather than atomically undoing the full operation

**Issues Deferred:**

- v2 layout schemes (horizontal, fan, tagged) — deferred to v2.0
- Error dialogs for empty selection — intentionally out of scope (fail silently)

**Technical Debt Resolved:**

- 3 audit-identified tech debt items resolved in final milestone commit (docs(v1.0))
