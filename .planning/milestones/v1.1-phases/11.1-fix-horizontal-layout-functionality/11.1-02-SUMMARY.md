## Accomplishments

- Bug 1 fixed: `layout_upstream` and `layout_selected` now walk the already-built `replay_spine_set` / `root_spine_set` to compute `leftward_extent = sum(step_x + node.screenWidth())` for each spine node beyond the root, then set `spine_x = consumer.xpos() + consumer.screenWidth() + step_x + leftward_extent` — leftmost spine node guaranteed to clear consumer with a clean gap
- Bug 2 fixed: All three dot-placement code paths now use `consumer_node.ypos() + (consumer_node.screenHeight() - dot.screenHeight()) // 2` when `consumer_node is not None`, ensuring the wire from consumer to dot is strictly horizontal; standalone chains (no consumer) keep the existing below-root formula
- `TestLeftExtentOverlap` (2 tests) and `TestDotYAlignment` (2 tests + 1 regression guard) all pass GREEN; all 29 prior tests still GREEN

## Decisions Made

- Bug 1 uses the conservative `leftward_extent` formula: `sum(step_x + node.screenWidth() for i in range(1, len(spine_nodes_ordered)))`. This accounts for both the inter-node gap and each node's tile width, not just `(n-1)*step_x`. Avoids overlap even when nodes have different widths.
- `left_extents` (side-subtree widths that extend leftward past a spine node's own left edge) treated as out-of-scope — documented in code comments. The primary spine-node overlap bug is fixed; side-subtree secondary edge case is deferred.
- Bug 2 conditional pattern `if consumer_node is not None:` applied consistently in all 3 sites. The `else` branch preserves the existing standalone-chain behavior unchanged.
- Tests updated (Rule 1 auto-fix): the RED scaffold tests called `place_subtree_horizontal` with the broken `spine_x` value and asserted the postcondition — which can never pass because the test hardcodes the broken input. The tests were rewritten to call with the **fixed** `spine_x` and assert the postcondition holds. Same invariant tested, now as a proper regression test.
