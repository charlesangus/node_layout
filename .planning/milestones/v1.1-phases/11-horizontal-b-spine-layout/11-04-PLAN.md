Two matching edits — one in `layout_upstream`, one in `layout_selected`.

**Pattern to add (the upstream walk):**
After reading `root_mode` from `root_stored_state`, insert an ancestor scan. If the selected root
does not itself carry `mode='horizontal'`, walk `input(0)` upstream until a node with
`mode='horizontal'` is found. If one is found, use that ancestor as the effective replay root for
the horizontal path. If none is found, fall through to the vertical path unchanged.

The walk must stop when `input(0)` returns `None` or when the current node's mode is not
`'horizontal'`. The first node found with `mode='horizontal'` becomes the new `root`.

**In `layout_upstream` (around lines 1185-1213):**

Replace the existing block:
```python
            root_stored_state = node_layout_state.read_node_state(root)
            root_mode = root_stored_state.get("mode", "vertical")

            replay_spine_set = None
            if root_mode == "horizontal":
```

with:
```python
            root_stored_state = node_layout_state.read_node_state(root)
            root_mode = root_stored_state.get("mode", "vertical")

            # If the selected node is downstream of a horizontal chain (its own mode is
            # vertical), walk input(0) upstream to find the first ancestor stored as
            # horizontal. That ancestor becomes the effective replay root.
            if root_mode != "horizontal":
                ancestor_cursor = root.input(0)
                while ancestor_cursor is not None:
                    ancestor_state = node_layout_state.read_node_state(ancestor_cursor)
                    if ancestor_state.get("mode") == "horizontal":
                        root = ancestor_cursor
                        root_mode = "horizontal"
                        break
                    ancestor_cursor = ancestor_cursor.input(0)

            replay_spine_set = None
            if root_mode == "horizontal":
```

Note: reassigning the local `root` variable here is safe — it is the variable used for the rest
of this iteration of the outer for-loop. Python for-loop rebinding is per-iteration; the outer
loop continues with the next element on the next pass regardless.

**In `layout_selected` (around lines 1310-1340):**

Apply the identical ancestor-walk insertion after the `root_mode` read, before the
`if root_mode == "horizontal":` branch. Use the same variable names (`ancestor_cursor`,
`ancestor_state`) for consistency.

The two insertions are structurally identical; only the surrounding function differs.
