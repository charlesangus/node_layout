## Tech Debt by Phase

### Phase 4: Preferences System (3 items)

**PREFS-02 — Requirements text conflict with implemented design**
- **What:** REQUIREMENTS.md says "SUBTREE_MARGIN significantly reduced from 300". `DEFAULTS['base_subtree_margin'] = 300` is unchanged from the old constant.
- **Why intentional:** At `reference_count=150` with `normal_multiplier=1.0`, the sqrt formula returns exactly 300 — deliberately backward-compatible. Small subtrees receive tighter spacing via sqrt scaling, but the base default is not reduced.
- **Resolution:** Update PREFS-02 text: "Subtree margin dynamically scales with node count (sqrt formula); base default is backward-compatible; effective spacing for small subtrees is smaller than 300."

**PREFS-04 — Cross-phase delivery, never formally re-verified**
- **What:** Phase 4 VERIFICATION declared NOT SATISFIED (preset commands deferred to Phase 5). Phase 5 delivered `layout_upstream_compact`, `layout_selected_compact`, `layout_upstream_loose`, `layout_selected_loose` — functionally satisfying PREFS-04. Phase 5 VERIFICATION claimed CMD-01, CMD-02, SCHEME-01 but not PREFS-04 explicitly.
- **Code status:** Fully wired (integration checker confirmed). `menu.py` has all three preset commands registered.
- **Resolution (choose one):**
  - Add a `note` field to Phase 4 VERIFICATION stating PREFS-04 was resolved by Phase 5 delivery
  - Add a line to Phase 5 VERIFICATION Requirements Coverage table for PREFS-04

**PREFS-06 — Dialog design deviation from requirement text**
- **What:** REQUIREMENTS.md says "Dialog exposes a preset selector and numeric fields". No `QComboBox` preset selector exists; tight gap multiplier field absent.
- **Why intentional:** Phase 4 CONTEXT.md locked: "Compact/Normal/Loose are Phase 5 menu commands, NOT a preset selector widget"; "tight gap multiplier removed — compact/loose/normal multipliers serve this role."
- **Resolution:** Update PREFS-06 text: "Dialog exposes 7 numeric QLineEdit fields for base_subtree_margin, scaling_reference_count, compact_multiplier, normal_multiplier, loose_multiplier, loose_gap_multiplier, and mask_input_ratio."
