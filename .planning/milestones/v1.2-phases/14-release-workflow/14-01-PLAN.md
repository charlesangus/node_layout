**Job 2: `build` (REL-03, REL-04):**
- `runs-on: ubuntu-24.04`
- `needs: test` (gates on test job passing)
- `permissions: contents: write` (required for GitHub Release creation)
- Step: Checkout with `actions/checkout@v4` (no `ref:` override -- default is correct for tag triggers)
- Step: Build ZIP with these exact shell commands:
  ```
  mkdir node_layout
  cp node_layout.py node_layout_state.py util.py \
     node_layout_prefs.py node_layout_prefs_dialog.py \
     menu.py make_room.py README.md LICENSE node_layout/
  zip -r node_layout-${{ github.ref_name }}.zip node_layout/
  ```
  The hardcoded file list is: `node_layout.py`, `node_layout_state.py`, `util.py`, `node_layout_prefs.py`, `node_layout_prefs_dialog.py`, `menu.py`, `make_room.py`, `README.md`, `LICENSE` (9 files total). Do NOT use globs.
- Step: Publish Release using `softprops/action-gh-release@v2` with:
  - `files: node_layout-${{ github.ref_name }}.zip`
  - `generate_release_notes: true`
  - Do NOT set `name:` (defaults to tag name, e.g. `v1.2`)
  - Do NOT set `draft: true` (release publishes immediately)

ZIP naming: `node_layout-${{ github.ref_name }}.zip` produces e.g. `node_layout-v1.2.zip` (keeps `v` prefix per user decision).
