Create the directory `.github/workflows/` and the file `.github/workflows/ci.yml` with exactly this content:

```yaml
name: CI

on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]

jobs:
  lint-and-test:
    runs-on: ubuntu-24.04

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install ruff pytest

      - name: Lint with Ruff
        run: ruff check .

      - name: Run tests
        run: pytest tests/ -v
```

Key design decisions (from CONTEXT.md locked decisions):
- **Trigger:** all branches on push + all pull requests (branches: ["**"])
- **Single job:** lint-and-test (not parallel jobs)
- **Sequential:** Ruff runs before pytest — if lint fails, tests don't run (GitHub Actions default: steps stop on failure)
- **Runner:** ubuntu-24.04 (explicit, currently what ubuntu-latest resolves to)
- **Python:** 3.11 only, no matrix (matches Nuke 16 bundled Python)
- **No `cache: 'pip'`:** Omitted per research — only 2 packages, caching adds complexity for <10s savings, and without a requirements file the cache key is unstable
- **No version pin on ruff:** Matches sibling project pattern; always installs latest

Do NOT add any other steps, jobs, or configuration. The workflow must be minimal.
