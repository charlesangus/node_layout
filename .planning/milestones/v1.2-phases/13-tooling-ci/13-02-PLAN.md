**Step 2: Fix remaining E501 violations in node_layout.py (~88 lines over 100 chars).**

The long lines fall into these categories — apply the appropriate wrapping strategy:

a) **Long function signatures** (e.g., `def compute_dims(node, x, y, memo, snap_threshold, ...)` at lines 857, 942, etc.):
   - Break after opening paren, one parameter per line, closing paren on its own line
   - Example:
   ```python
   def compute_dims(
       node, memo, snap_threshold, node_count,
       node_filter=None, scheme_multiplier=None,
       per_node_h_scale=None, per_node_v_scale=None,
   ):
   ```

b) **Long list comprehensions** (e.g., `child_dims = [compute_dims(inp, memo, ...) for ...]` at lines 875, 885, 1020):
   - Break after opening bracket, wrap the expression and for-clause
   ```python
   child_dims = [
       compute_dims(inp, memo, snap_threshold, node_count, ...)
       for slot, inp in input_slot_pairs
   ]
   ```

c) **Long arithmetic/conditional expressions** (e.g., lines 158, 669, 904, 910):
   - Break at operators, indent continuation lines by 4 spaces
   ```python
   effective_margin = int(
       base * mode_multiplier * math.sqrt(node_count)
       / math.sqrt(reference_count) * font_mult
   )
   ```

d) **Long comments** (e.g., lines 898, 899):
   - Reflow to multiple comment lines under 100 chars each

e) **Lines with existing `# noqa: E501`** (line 938):
   - Keep the noqa comment if the line is a memo key that would be unreadable when split. Otherwise, wrap and remove the noqa.

f) **Long dict/list literals or function calls with many args**:
   - One argument per line with trailing comma

**Important:** Do NOT change any logic. Only change line breaks and indentation. Preserve all `# noqa` comments that are intentional. After wrapping, ensure the indentation level is correct for the surrounding block.
